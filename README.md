# Проекты системы

- `Admission.Auth` — сервис аутентификации и авторизации: регистрация, вход, refresh/access токены, управление учетными записями сотрудников и администраторов.
- `Admission.Dictionary` — сервис справочников приемной кампании: факультеты, программы, уровни образования и типы документов, а также импорт этих данных.
- `Admission.MailManager` — сервис обработки почтовых сообщений и уведомлений; принимает события через брокер, формирует и отправляет письма, хранит и обрабатывает inbox.
- `Admission.Api` [Unimplemented] — основной API предметной области поступления: профили абитуриентов, заявления, документы, программы и работа менеджеров с заявками.
- `Admission.Administration.Mvc` [Unimplemented] — административное веб-приложение для работы администратора с системой и служебными операциями. Сейчас в репозитории это минимальная заготовка.
- `MailContracts` — общая библиотека контрактов сообщений для интеграции между сервисами через брокер.

Вместе эти проекты образуют backend-систему приемной кампании. Инфраструктурные зависимости поднимаются через `docker-compose`: `PostgreSQL`, `RabbitMQ`, `MinIO` и `MailDev`.

> Также в папке `docs` если черновые диаграммы компонентов системы и архитектуры проекта Admission.Api

# Запуск

1. Надо склонировать репозиторий
2. Заходим в /Placeholder-Task-4472
3. Сам запуск:
   - Если есть докер - `docker compose up --build -d`
   - Если нету - то запустить по идее возможно, но я не гарантирую что сервисы найдут друг друга
4. Открыть ссылки которые будут ниже, и начать проверку системы
   - Auth swagger - http://localhost:5279/swagger/index.html
   - Dictionary swagger - http://localhost:5280/swagger/index.html
   - Mail Dev - http://localhost:1080

## ВАЖНО!!!
докер требудет что следующие порты были незаняты: 
- `5279` - Auth API
- `5280` - Dictionary API
- `5278` - Mail Manager API
- `1080` - MailDev Web UI
- `1025` - MailDev SMTP
- `5672` - RabbitMQ AMQP

- `5432` - PostgreSQL
- `6379` - Redis
- `5540` - Redis Insight

- `9001` - MinIO Console
